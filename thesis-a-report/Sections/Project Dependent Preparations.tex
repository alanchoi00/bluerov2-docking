\section{Project Dependent Preparations}

\subsection{Hardware Preparation}
\begin{itemize}
    \item Platform: BlueROV2 configuration, thruster layout and capabilities
    \item Camera selection: low-light sensitivity, global shutter preferred, underwater housings
    \item Lighting: directional LEDs, diffuse flood lights, controllable intensity
    \item Dock fixtures and markers: fiducial panels, LED beacons, mechanical capture hardware
    \item Tether management: reel, strain relief, quick-disconnects for safety tests
\end{itemize}

\subsection{Software and Stacks}
\begin{itemize}
    \item Middleware: ROS2 for nodes, topics and launch management
    \item Vision libraries: OpenCV, apriltag/ArUco libraries, PyTorch/TensorFlow for CNN models
    \item Language choices: Python for rapid prototyping, C++ for performance-critical nodes
    \item Logging and playback: rosbag2, structured experiment logging, dataset capture tools
\end{itemize}

\subsection{Tank and Pool Testing Plans}
\begin{itemize}
    \item Controlled tank experiments for repeatable turbidity levels and platform motion
    \item Test matrix: turbidity levels, lighting conditions, dock motion profiles, approach trajectories
    \item Safety and recovery plan for each test (tethered tests first, then untethered)
    \item Data capture plan: synchronized camera, IMU, and ground-truth (external tracking if available)
\end{itemize}

\subsection{Safety Considerations}
\begin{itemize}
    \item Electrical isolation and waterproofing verifications for all onboard electronics
    \item Pool compliance, trained personnel, and emergency stop procedures
    \item Safe handling of turbidity agents and environmental disposal per lab policies
    \item Mechanical safeguards for docking engagement to avoid collisions or entrapment
\end{itemize}

\subsection{Risk Assessment}
\begin{itemize}
    \item Vision failure due to extreme turbidity or lighting — impact and mitigation
    \item Tether entanglement or unexpected loads on the ROV during approach
    \item Hardware failure: camera housing leaks, thruster faults
    \item Data loss: logging failures or time-synchronisation issues
\end{itemize}

\subsection{Ethics Statement}
\begin{itemize}
    \item No live animals involved; tests confined to laboratory pool and controlled environments
    \item Use of non-toxic turbidity agents and adherence to waste disposal regulations
    \item Data privacy: no personally identifiable information collected
    \item Adherence to UNSW and lab safety and ethics guidelines
\end{itemize}

\subsection{Contingencies and Fallback Methods}
\begin{itemize}
    \item Fallback to active LED beacons or acoustic homing if vision quality degrades
    \item Conservative approach strategies and larger acceptance radii for final engagement
    \item Simulation and hardware-in-loop testing if field tests are delayed
    \item Modular system design to swap detectors or sensors without major rework
\end{itemize}
